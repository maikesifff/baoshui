自报税 软件用户手册

版本：1.0    更新日期：2025年3月

本文档为自报税软件的正式用户手册，用于说明软件的运行环境、功能、操作步骤及注意事项。用户与管理员可据此正确使用本系统。

第1章 引言

1.1 编写目的

本手册面向自报税软件的用户与管理员，说明软件的运行环境、功能、操作步骤及注意事项，便于用户正确使用本系统完成记账报税相关业务。

1.2 背景

自报税是一款面向小微企业及个体工商户的记账报税信息化软件，提供用户端（Web/Android App）与后台管理端，支持企业绑定、税局/个税信息配置、套餐开通、通知与客服等功能，旨在降低用户办税成本、提升办事效率。

1.3 定义与缩略语

  • 用户端：指面向最终用户的前台界面，包括浏览器网页与 Android 应用。
  • 后台/管理端：指面向运营人员的后台管理界面，用于用户管理、兑换码、通知等。
  • API：应用程序接口，前后端通过 HTTP 接口交换数据。
  • HTTPS：基于 TLS 的安全通信协议，用于保护用户数据传输。
  • WebView： Android 系统内置的网页渲染组件，用于在 App 内展示网页内容。

1.4 参考资料

自报税隐私政策、自报税软件说明书（若有）。

第2章 运行环境

2.1 硬件环境

服务器：x86-64 架构服务器或云主机，建议内存 2GB 及以上，硬盘 10GB 可用空间。

用户端（PC）：Intel/AMD x86-64 兼容计算机，内存 4GB 及以上。

用户端（手机）：Android 7.0 及以上，内存 2GB 及以上；或支持现代浏览器的 iOS/其他设备。

2.2 软件环境

服务器：Linux 操作系统（如 CentOS 7+、Ubuntu 18.04+）；Python 3.8+；SQLite 3；可选 Nginx 作为反向代理。

用户端：Chrome、Edge、Safari 等现代浏览器；或 Android 系统内置 WebView。

管理端：使用 PC 浏览器访问后台地址。

2.3 网络要求

服务器需具备公网 IP 或域名，并配置 HTTPS（推荐）。用户端与管理端需能访问服务器域名及相应端口。

第3章 安装与部署

3.1 服务端部署

1. 安装 Python 3.8 及以上、pip。

2. 将项目代码部署至服务器目录（如 /opt/baoshui）。

3. 安装依赖：pip install flask（或按项目 requirements.txt 安装）。

4. 配置环境变量（可选）：数据库路径、密钥、管理员密码等。

5. 使用 gunicorn 或 systemd 启动应用，并可选配置 Nginx 反向代理与 HTTPS。

6. 确保 data 目录存在且可写，用于 SQLite 数据库及上传文件。

3.2 用户端访问

浏览器：在 PC 或手机浏览器中访问部署后的域名及路径（如 https://example.com/baoshui/）。

Android App：安装自报税 Android 安装包（APK）后，应用内通过 WebView 加载上述地址，无需单独部署前端资源。

3.3 后台管理访问

在 PC 浏览器中访问后台地址（如 https://example.com/baoshui/admin.html），使用管理员密码登录。

第4章 用户端功能说明

4.1 注册与登录

用户可通过手机号或自定义账号注册；支持验证码登录与密码登录两种方式。注册时需同意《隐私政策》并勾选复选框，否则无法完成注册或登录。登录成功后系统会记录登录状态，并可根据是否已有企业自动选择或提示选择企业。

验证码登录：输入手机号或账号后点击「获取验证码」，收到短信后填入验证码即可登录。

密码登录：输入账号与密码，点击登录。若未注册，需先完成注册流程。

4.2 主页与待办

未登录时主页展示产品介绍、套餐价格、自报税使用说明等落地页。已登录时主页展示当前选择的企业、我的待办（如填写税局登录信息、选择记账方式、税务 Ukey、个税登录信息）、我的通知、记账/审账/报税节点说明及开通正式版入口等。待办项根据企业已保存的设置显示「立即处理」或「修改」，点击进入对应设置页。

记账/审账/报税节点：主页中部展示三个业务节点，分别对应不同时间段。记账节点显示上月末至本月初；报税节点显示本月初后的申报期。点击节点可切换查看对应说明。

4.3 企业与管理

用户可绑定多个企业；在主页顶部可切换当前企业。「我的企业」入口可进行企业的增删改；当前选中的企业不可删除。企业相关数据（如税局信息、个税信息、记账方式等）均按企业维度存储。

添加企业：在「我的企业」页点击「添加企业」，输入企业名称后保存。

删除企业：在「我的企业」列表中选中非当前企业，点击删除并确认。当前选中的企业不可删除。

4.4 税局与个税设置

税局登录信息：用户可填写税局网站、登录方式、法人身份证、法人密码等信息并保存，用于后续代理报税或指导。填写项包括：税局网站地址、登录方式（CA 证书/账号密码等）、法人身份证号、法人登录密码等。

个税登录信息：可填写个税账号与密码并保存，供个税申报使用。

记账方式：在对应页面选择自记账、代理记账等类型并保存。

税务 Ukey 或税盘：在对应页面选择或填写 Ukey/税盘信息并保存。

4.5 套餐与兑换码

用户可在「开通正式版」相关页面查看套餐说明与价格。支持通过兑换码兑换套餐时长；输入有效兑换码并选择当前企业即可开通。未选择企业或未登录时，会提示先登录并选择企业。

套餐类型：小规模纳税人、一般纳税人等，不同套餐对应不同记账类型与时长。

兑换码使用：在开通正式版页面输入兑换码，选择企业后点击兑换。兑换成功后该企业即获得对应套餐时长。

4.6 报表与发现

报表相关页面用于展示与用户/企业绑定的报表数据（需登录并选择企业）。发现页展示运营配置的发现条目，可点击跳转至指定链接。

4.7 聊天与客服

用户可在聊天页查看与客服的对话列表，进入会话后可发送消息。未登录时无法使用聊天功能。后台管理员可主动向用户发起对话。

4.8 我的与设置

「我的」页展示用户信息、已绑定手机号、我的企业入口、设置入口、退出登录等。设置页可查看关于信息、隐私政策链接等。隐私政策页可单独在浏览器中打开；若带参数（如 from=app）打开，则显示标题栏与「返回主页」按钮。

第5章 后台管理功能说明

5.1 登录与导航

管理员在后台登录页输入密码登录，登录成功后可使用顶部导航在各功能模块间切换。导航包括：用户管理、兑换码、优惠券、通知、发现、客服配置、退出等。

5.2 用户管理

用户列表展示所有注册用户的手机号/账号、注册时企业名、企业数量、注册时间等。支持手动创建用户（账号、密码、可选企业名）。点击某用户可进入用户详情页，查看该用户基本信息、企业列表、企业设置（税局/个税等）、会话数，并可重置该用户登录密码、发起客服对话。

创建用户：在用户列表页点击「创建用户」，填写账号、密码、可选企业名后保存。

重置密码：在用户详情页点击「重置登录密码」，输入新密码并确认后保存。

5.3 兑换码管理

可创建兑换码，指定纳税类型、记账类型、开通时长等。可查看兑换码列表及使用状态，删除未使用的兑换码。兑换码为一次性使用，兑换后即失效。

5.4 通知管理

可发送通知：填写标题、内容，可选指定手机号（留空则全员）。可查看通知列表，按手机号筛选，并对单条通知进行编辑、删除。

5.5 发现管理

可添加发现条目（标题、链接、排序）。可对已有条目进行编辑、删除。发现条目用于在用户端「发现」页展示，点击可跳转至指定 URL。

5.6 客服配置

可配置客服微信号及说明文案，供用户端展示。用户点击客服相关按钮时可看到微信号及联系方式。

第6章 操作流程示例

6.1 用户首次使用

1. 打开自报税网页或 App，阅读隐私政策并同意。

2. 点击注册/登录，完成验证码或密码登录。

3. 若有提示，填写或确认公司名称。

4. 在主页选择或添加企业。

5. 按待办提示依次完成税局登录信息、记账方式、个税信息等设置。

6. 如需正式版，可通过兑换码或后续支付流程开通。

6.2 管理员发送通知

1. 登录后台，进入「通知」。

2. 填写标题与内容，可选填指定手机号。

3. 点击发送；可在下方列表中查看、编辑或删除历史通知。

6.3 管理员创建兑换码

1. 登录后台，进入「兑换码」。

2. 选择纳税类型、记账类型、开通时长，生成兑换码。

3. 将兑换码提供给用户，用户在用户端输入即可兑换。

第7章 安全与隐私

用户密码经加密存储；通信建议使用 HTTPS。用户可随时在设置或隐私政策页查看《隐私政策》；注册/登录前需勾选同意。企业敏感信息（如税局密码、个税密码）仅在用户本人及授权管理员可见，请妥善保管账号与密码。

第8章 常见问题

Q：无法登录？ A：请确认账号与密码正确，或使用验证码登录；若为首次使用请先注册。

Q：待办项不显示「修改」？ A：请确认已登录并已选择企业；选择企业后页面会刷新待办状态。

Q：兑换码无效？ A：请确认兑换码未过期、未使用，且当前已选择企业。

Q：后台忘记密码？ A：需在服务器环境变量或配置中重置管理员密码后重启服务。

Q：如何切换企业？ A：在主页顶部点击企业名称，展开下拉列表后选择目标企业。

Q：隐私政策在哪里查看？ A：登录/注册页有链接，或设置页、隐私政策页均可查看。

第9章 附录

附录A 接口说明（概要）

用户端与服务器通过 RESTful 风格 API 通信，主要包含：

  • 认证：注册、登录、验证码发送
  • 用户与企业：企业列表、添加/删除企业、企业设置
  • 企业设置：税局信息、个税信息、记账方式、Ukey 等
  • 套餐与兑换：套餐状态、兑换码兑换
  • 通知、发现、聊天、后台管理等模块

具体接口路径与参数以实际代码与接口文档为准。

附录B 版本历史

V1.0：初始版本，包含用户端与后台管理端主要功能，支持 Web 与 Android App。

附录C 注意事项

1. 企业敏感信息（税局密码、个税密码等）请妥善保管，切勿泄露。

2. 兑换码为一次性使用，兑换后即失效，请勿重复使用。

3. 建议使用 HTTPS 访问，确保数据传输安全。

4. 若长时间未操作，登录状态可能过期，需重新登录。

5. 后台管理员需妥善保管登录密码，定期更换。

附录D 数据存储说明

用户数据、企业数据、企业设置等均存储在服务器端 SQLite 数据库中。用户密码经 PBKDF2 等算法加密存储。会话信息通过 token 机制维护，登录后 token 有效期内可访问需认证的接口。
