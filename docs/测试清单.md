# 自报税 - 测试清单

以下按模块分组，测试时可按「组」或「条」指定范围执行。未要求测试时仅保留清单，不执行。

**测试约定：**

- **联动项（分阶段 + 记录，省 token）**：涉及「后台操作 → 前台有反应」或「前台操作 → 后台有反应」的项，**不要求同时开两个页面**。先测一端（如先测后台），把本次操作的**关键数据**记入下方「联动操作记录」对应行；等测另一端（如前台）时，根据记录里的内容去核对是否一致即可。这样避免双页同时开着带来的 token 消耗。
- **需协助则跳过**：依赖用户本人配合的项（如验证码登录需本人收短信、注册需收验证码等），测试时**直接跳过**，并在结果处标记为 **⏭ 跳过（需用户协助）**，不阻塞清单执行。

**联动操作记录（先测一端时填写，测另一端时对照）：**


| 联动对                    | 先测端操作/数据记录（测完后填写）                    | 另一端验证时对照                                    |
| ---------------------- | ------------------------------------ | ------------------------------------------- |
| 11.5 客服配置 → 3.3.3 客服微信 | 例：微信号改为 xxx，说明改为 xxx                 | 前台点「客服微信」是否显示上述内容                           |

| 11.1 兑换码 → 8.1.2 订单页兑换 | 例：创建兑换码 xxxxxx，套餐/时长 xxx             | 前台订单页输入该码兑换是否成功                             |

使用说明：  

- 指定组：如「测试 三、前台 - 已登录主页」。指定条：如「测试 3.2.1、3.2.2、3.2.3」。  
- 联动项（省 token）：标有 **【联动】** 的项不要求同时开两页。先测一端（如后台 11.3 发通知），在顶部「联动操作记录」表对应行填写本次操作的关键数据；测另一端（如前台 3.4.1 我的通知）时，根据记录核对是否一致。结果仍记在对应条后（✅ / ❌），并注明是否按记录完成对照。  
- 需协助则跳过：标有 **【需用户协助则跳过】** 的项（如验证码登录、注册收验证码），不执行实际操作，直接标记 **⏭ 跳过（需用户协助）**。
